User Manual.

Dashboard (Overview)

This is the main screen that welcomes you when the application starts. Here you can track the summary of your current tasks, their real-time status, and progress.

Features

  • Task List: All tasks you create are listed as cards.
  • Status Indicators: The status of each task (Ready, Running, Completed, Error) is shown with colored labels.
  • Progress Bar: Visually indicates how much of the task has been completed.
  • Quick Actions: You can stop (Pause) or resume (Resume) tasks directly from this screen.

Task Management

This is the most comprehensive section where you create new automation tasks and edit existing ones. You can open a new task form with the "+ Add Task" button.

Form Fields and Details

Task Name The name you give to identify the task. It appears with this name in lists (e.g., "Daily SEO Traffic").
Target URL The main website address where the automation will be performed. (e.g., https://example-site.com).
Navigator Type Determines how to reach the target site.
  • Direct URL: Goes directly to the address.
  • Google Search Fake: Goes to the address as if it came from a Google Search result, but does not actually go through a Google Search result, nor does the target URL need to exist in any search result. This is for Analytics manipulation; Analytics defines a vast majority of visits sent with this module as "Organic Search".
  • Google Search Direct: Goes to the address directly from the Google Search Result page containing the target URL. This means you need to specify the SERP URL where the target URL is located in this module. The module starts the task directly from this URL and performs a click action on the search result. Clicks are reflected in Google Search Console. This is the MOST EFFICIENT NAVIGATOR MODULE FOR SERP MANIPULATION purposes. Because unlike the Google Search module, it does not have a long process and provides an advantage by performing maximum Google clicks per unit of time.
  • Google Search: Finds the site by searching on Google and clicks on it. Clicks are reflected in Google Search Console, however, there is no experience or information suggesting it has an advantage over "Google Search Direct" for the purpose of increasing ranking on Google; on the contrary, it is more inefficient than Google Search Direct as it extends the process and reduces the number of visitors clicking on the search result per unit of time.
  • Referrer Spoof Navigation: Accesses the target URL with referrer information. (e.g., "https://www.facebook.com/"). This is for Analytics manipulation; it does not actually go to the referrer site, but tells traffic analyzers like Analytics that it came to this site from the referrer site.
Mission Type Determines what to do after reaching the site.
  • Just Visit:
    Function: Opens the page, loads resources (images/css), and simulates mouse movements to appear as an organic visitor.
    Wait Time: Spends a random amount of time between 3 and 8 seconds after the page loads.
    Average Total Duration: Page load time + ~5.5 seconds.
  • Surfing (Simple):
    Function: Performs random Scroll, Engage (Mouse/Focus), and Idle actions in a dynamic loop on the page.
    Duration Target: A total duration target is randomly determined in the code between 10 and 20 seconds.
    Average Total Duration: Page load time + ~15 seconds.
  • Surfing (Advanced):
    Function: Follows a 9-step scenario containing much more complex and human-like behaviors: Wait & Jitter, Focus on Headings/Texts, Wait Again, Scroll, Focus Again, Focus on Form Elements if any, Scroll and Jitter Combination, Click Internal Links (Navigation), Scroll and Wait on New Page.
    Duration: The sum of all these steps (excluding navigation and loading) takes approximately 40-60 seconds.
    Average Total Duration: Page load times + ~50 seconds.
  • Cookie Generator:
    Function: Clicks on random links within the same domain and navigates purely for the purpose of collecting cookies.
    Page Count (Steps): Navigates a random number of pages between 8 and 12 (Average 10 pages).
    Average Total Duration: (10 Pages x Load Time) + (10 Pages x ~3.5 Seconds Wait) = 35 seconds + Total Load Times.
    Note: Cookies exported with this module are not sufficient for the success of Google Search and Google Search Direct (these two modules used for SERP manipulation can only be successful with cookies taken from a real browser), but they may be used on various platforms like Youtube, Twitter, etc.
Select Proxy Determines which IP address the connection will be made through.
  • Without Proxy: Uses your own IP address (Not recommended).
  • Any (All): Randomly selects one from all registered proxies.
  • Any (Singles): Randomly selects from proxies that are not included in a specific group.
  • Group: Randomly selects from all proxies within the selected group.
  • Specific: Selects a single proxy. If a rotating proxy is to be used, it should be added as a single proxy and selected as specific.
Accept Language The language preference the browser will report to the site. By default, it is set to Turkish as the primary language and English as the secondary language (tr-TR,tr;q=0.9...). You can change it according to the target location and the language of that location.
Cookies Import / Export
  • Import: Browses the internet with browser history by loading previously saved cookies. You need to select a folder.
  • Export: Saves the cookies created after the process to a file. Import is not mandatory for Export, but if you select both import and export in a task, your imported cookies will be updated, their lifespan extended, and their effect increased.
  • Restore on Failure: Used cookie files are moved to a folder named 'processed' automatically created inside the folder specified as the import folder. Thus, the same cookie file cannot be used by more than one session in the same task. However, to prevent cookie waste in sessions that fail due to proxy issues etc., you can enable the “Restore Cookies on Failure” feature. Thus, cookies imported by failed sessions will be re-included in the import folder, at the end of the cookie import queue.
Resource Optimization When enabled, it speeds up the process and saves bandwidth by preventing the loading of unnecessary resources (images, fonts, ad scripts, etc.). You can customize the types to be blocked. Those added to the Exception List in the domain:resource format will be exempt from blocking rules to the extent you allow. To exclude all resource types of a domain from the blocking list, add it to the Exception List as domain.com:all. If you only want scripts to run, add it as domain.com:script. Or you can allow multiple resource types with the format domain.com:script,fetch.
Target Sessions The total number of transactions that this task must complete successfully. For example, if you enter "100", the system runs until it makes 100 successful visits.
Time (Hours) Important Feature: Determines how many hours the total process will be spread over.
For example: If you enter 100 Sessions and 1 Hour, the system spreads them homogeneously over 1 hour instead of finishing 100 sessions immediately (approximately one transaction every 36 seconds).

Proxy Manager (Proxies)

This is the section where you manage the IP addresses to be used in automation processes. Using a proxy ensures that transactions appear to come from different users and reduces the risk of being blocked.

Features

  • Add Proxy: Allows you to add proxies one by one. Format: ip:port:user:pass or just ip:port.
  • Bulk Add: Allows you to add a large number of proxies in bulk. You can paste them with one proxy per line.
  • Grouping: You can categorize proxies (e.g., "Mobile Proxies", "USA Proxies"). You can select these groups when creating a task.
  • Check: You can test whether the added proxies are working. Non-working ones are automatically set to passive status.

Settings

This is the section containing global settings that affect the general behavior of the application.

Max Concurrent Sessions Determines how many browser windows (processes) will run on your computer at the same time. You should adjust it according to your computer's performance (RAM/CPU). High values may slow down the system.
Max Consecutive Failures Determines after how many consecutive failed sessions in a task the task will be stopped. For example, if you enter "500", the task stops after 500 consecutive failed sessions. You should set this value to prevent running forever in cases such as the target page being unreachable or the proxy limit running out.
Max Retry Attempts The number of retry attempts for failed transactions (specifically affects mechanisms where the number of attempts is not specified in the code). For example, if it fails for various reasons in the action of clicking the target URL in the Google Search Result, it retries this many times instead of giving an error and terminating the task. When the number of attempts is over, it terminates the process and marks it as failed.
License Management This is the area where you can enter your license key and see the status and expiration date of your active license.
Timeouts
  • Request Timeout: Maximum time to wait for a web request to respond.
  • JSRenderer Load Timeout: Time to wait for the page to load completely.
  • Ping Timeout: Maximum wait time for the ping process to complete in modules using the ping process.
  • Proxy Connection Timeout: Proxy connection check duration.
  • Base Delay: The time to wait between transactions at the Engine level, in transactions where the wait time is not specifically specified. For example, in accordance with the "Max Retry Attempts" limit, "Base Delay" is waited between retry attempts for failed transactions.
